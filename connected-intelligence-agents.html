Three layers. One agentic mesh.
The Orchestration Agent sits at the top, routing every request and carrying context. Below it, Platform Agents handle the intelligence work that happens before any response reaches a client. Digital Channel Agents own every client-facing surface. LOB Agents own product knowledge and domain decisions. Platform and LOB agents with no direct client-facing surface are headless agents — they run via API or event, invisible to the end user.

Governance & Trust layer
For regulated industries, governance is not a policy document — it is an architectural property. Four governance concerns are designed into the mesh: determinism, human oversight, observability, and identity. The escalation path below shows how probabilistic agent outputs are routed based on confidence before reaching a client.
Deterministic shells / probabilistic coresLLMs reason and narrate — they never calculate, never execute. Deterministic rules govern what they can produce.
DataEnvelope pattern

Every agent that handles financial data receives a structured envelope from the data layer. The LLM narrates; it never does arithmetic. Numbers are deterministic outputs, not model outputs.

Bounded output contracts

Each Platform and LOB agent has a defined output schema. The model cannot produce a response outside that schema — confidence is always explicit, format is always controlled.

Action firewall

No LLM output can directly execute a payment, FX transaction, or wire. All actions are staged by a deterministic layer and require either human approval or a governed agent scope token.

Human-in-the-loop (HITL) & human-on-the-loopHigh-confidence outputs are delivered; low-confidence outputs are escalated. No silent failures.
Confidence-based routing

The Critique Agent scores every response before delivery. Below a defined confidence threshold, the response is either escalated for human review or delivered with an explicit uncertainty signal.

Human-in-the-loop

For consequential actions — wire approvals, FX execution, large sweeps — a human confirmation step is required regardless of agent confidence. The system cannot override this.

Human-on-the-loop

The Supervisor Agent monitors all agent activity. Anomalous behaviour, drift, or unexpected outputs trigger an alert to a named human overseer — before they affect the client.

Agent observabilityEvery call, cost, latency, and reasoning step is traced. No black boxes.
Full reasoning trace

Every LLM call, tool invocation, and agent handoff is logged with its inputs, outputs, latency, and token cost. The SR 11-7 model risk log is a by-product of normal operation, not a separate compliance exercise.

Drift detection

The Observability Agent continuously monitors agent outputs for deviation from baseline behaviour. Drift triggers an alert to the Supervisor Agent before a client is affected.

Cost attribution

Every interaction is attributed to a channel, a LOB agent, and a client segment. The economics of the agentic mesh — token cost, latency, ROI — are visible in real time.

Identity, registry & least-privilegeEvery agent has a registered identity. Every call carries only the permissions it needs.
Agent registry

Every agent — internal and external — is registered with a unique identity, capability declaration, owner, and permission scope. No unregistered agent can participate in the mesh.

Delegated scope tokens (OAuth2 / RFC 8693)

Third-party agents (TMS, ERP, corporate AI) operate only within the scope the client has explicitly delegated. The Orchestration Agent enforces the ceiling; the token carries the proof.

Least-privilege execution

Each agent receives only the data and tools it needs for the specific task. The FX Agent cannot read wire entitlements; the Payments Agent cannot read FX positions. Isolation is enforced, not assumed.
